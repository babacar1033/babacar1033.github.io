Artifact Overview

Title

Generative AI LLM Infrastructure: Training Process and Costs — A Visual Framework

Introduction

Training advanced generative AI large language models — such as GPT-4, LLaMA-3, Gemini Ultra, and Claude 3 Opus — requires massive infrastructure investments spanning data acquisition, GPU/TPU compute clusters, energy consumption, and months of continuous operation. For AI practitioners, product managers, and investment decision-makers, understanding where these costs originate and how they scale is no longer optional. This artifact demystifies the LLM training lifecycle through visual analysis and real-world cost data.

Description

This framework walks through the seven-stage LLM training pipeline (data collection → preprocessing → tokenization → pre-training → fine-tuning → RLHF → deployment), maps four cost pillars (Data, Compute, Energy, Time), and provides a side-by-side comparison of infrastructure choices and estimated training costs across four leading foundation models.

Objective

To clearly communicate the end-to-end LLM training process; identify and quantify the four primary cost categories (data, compute, energy, time); compare infrastructure choices across leading foundation models; and present findings in a visually compelling, portfolio-quality format accessible to both technical and non-technical audiences.

Process

I reviewed published research papers, technical reports, and cost analyses from OpenAI, Meta, Google DeepMind, and Anthropic, supplemented by public compute cost estimates from SemiAnalysis and Epoch AI. I mapped the training pipeline stages, categorized cost drivers, and synthesized infrastructure comparisons into an interactive visual framework built with HTML, CSS (Tailwind), and JavaScript.

Tools & Technologies Used

HTML5 / SVG CSS3 / Tailwind JavaScript (Vanilla) GitHub Pages Epoch AI / SemiAnalysis

Value Proposition

Unique Value: Most discussions of LLM training costs focus on a single dimension — usually compute. This artifact uniquely integrates all four cost pillars (data, compute, energy, time) into a unified visual framework, enabling stakeholders to understand the full cost structure before committing to infrastructure decisions. The side-by-side model comparison also highlights how architectural choices (parameter count, context window, training tokens) translate directly into dollar costs.

Relevance to Audience: For hiring managers in AI/ML, this artifact demonstrates cost literacy — a critical competency for roles in AI infrastructure, MLOps, and strategic AI leadership. For academic evaluators, it directly addresses AIML-501 learning outcomes: communicating training processes, analyzing infrastructure requirements, and quantifying resource utilization.

The LLM Training Pipeline

Training a large language model is a seven-stage industrial process spanning months of continuous compute. Each stage has distinct infrastructure requirements, bottlenecks, and cost profiles. Click any stage to expand its details.

1

Data Collection & Acquisition

Cost pillar: Data

Web crawls, licensed datasets, curated corpora

Sources
  • ▸ Common Crawl (petabytes of web text)
  • ▸ Books3, Project Gutenberg
  • ▸ GitHub code repositories
  • ▸ Wikipedia, ArXiv, PubMed
  • ▸ Licensed publisher datasets
Infrastructure
  • ▸ Distributed web crawlers
  • ▸ Petabyte-scale object storage (S3/GCS)
  • ▸ High-bandwidth data pipelines
  • ▸ Deduplication at scale
Cost Drivers
  • ▸ Storage: ~$20–50K/PB/month
  • ▸ Data licensing: $1M–$50M+
  • ▸ Crawl compute: hundreds of nodes
  • ▸ Human annotation: $5–25/hour
2

Data Preprocessing & Filtering

Cost pillar: Compute + Time

Quality filtering, deduplication, PII removal, language detection

Key Steps
  • ▸ MinHash deduplication
  • ▸ Quality scoring (perplexity filters)
  • ▸ Toxicity & adult content filtering
  • ▸ Near-dedup at doc & paragraph level
  • ▸ Domain & language balancing
Scale
  • ▸ Input: 50–500TB raw HTML
  • ▸ Output: 1–15TB clean text
  • ▸ Retention rate: 2–30%
  • ▸ Runs: days to weeks on 1K+ CPUs
Tools
  • ▸ Apache Spark / Ray
  • ▸ CCNet, DataComp, Dolmino
  • ▸ fastText language detection
  • ▸ Regex & heuristic filters
3

Tokenization

Cost pillar: Time

BPE or SentencePiece — converting text to integer token sequences

Tokenizer Approaches
  • ▸ BPE (GPT-4): 100K vocabulary, handles code well
  • ▸ SentencePiece (Gemini): language-agnostic unigram model
  • ▸ LLaMA tokenizer: modified BPE with 32K vocab
  • ▸ Context window impacts total token count
Scale & Speed
  • ▸ GPT-4: ~13T training tokens
  • ▸ LLaMA-3 70B: ~15T tokens
  • ▸ Tokenization: ~1–5 GB/s throughput
  • ▸ Output stored as binary shards (MMap)
4

Pre-Training

Cost pillar: Compute + Energy (dominant)

Next-token prediction on trillions of tokens across thousands of GPUs

Objective

Minimize cross-entropy loss on next-token prediction across the full training corpus. The model learns grammar, facts, reasoning, and world knowledge purely through self-supervised learning — no human labels required at this stage.

Infrastructure
  • ▸ 1K–25K NVIDIA A100/H100 GPUs
  • ▸ NVLink + InfiniBand (400 Gb/s)
  • ▸ Tensor / pipeline parallelism
  • ▸ bf16 mixed-precision training
  • ▸ Gradient checkpointing
Cost & Duration
  • ▸ 60–90% of total training cost
  • ▸ 2–6 months continuous run
  • ▸ A100 GPU: ~$2–4/hr (cloud)
  • ▸ H100 GPU: ~$3–6/hr (cloud)
  • ▸ Mid-run failures = catastrophic
5

Supervised Fine-Tuning (SFT)

Cost pillar: Data + Compute

Training on curated instruction-response pairs to align behavior

Process
  • ▸ 10K–1M human-written instruction/response pairs
  • ▸ Format: chat templates (system + user + assistant)
  • ▸ Domains: coding, math, reasoning, safety
  • ▸ Duration: hours to days (much cheaper than pre-training)
  • ▸ Transforms base model → instruction-following model
Cost
  • ▸ Human annotators: $1M–$20M
  • ▸ Compute: 1–5% of pre-training cost
  • ▸ Data quality > data quantity here
  • ▸ OpenAI used ~40 trainers for InstructGPT SFT
6

RLHF — Reinforcement Learning from Human Feedback

Cost pillar: Data + Compute

Reward modeling + PPO to align with human preferences

Reward Model (RM)
  • ▸ Trained on human preference rankings
  • ▸ Rankers compare model outputs A vs B
  • ▸ RM learns to score helpfulness & safety
  • ▸ ~100K–500K comparison pairs
PPO Training
  • ▸ Policy (LLM) generates rollouts
  • ▸ RM scores each rollout
  • ▸ KL-divergence penalty keeps model near SFT
  • ▸ Most computationally complex stage
Modern Alternatives
  • ▸ DPO (Direct Preference Optimization)
  • ▸ RLAIF (AI-generated feedback)
  • ▸ Constitutional AI (Anthropic)
  • ▸ 2–5× cheaper than standard RLHF
7

Deployment & Inference Infrastructure

Cost pillar: Compute (ongoing)

Serving the model at scale — often the dominant lifetime cost

Serving Stack
  • ▸ vLLM / TGI for throughput optimization
  • ▸ PagedAttention for KV cache efficiency
  • ▸ Quantization (INT8, AWQ, GPTQ)
  • ▸ Speculative decoding
  • ▸ Multi-region load balancing
API Pricing (per 1M tokens)
  • ▸ GPT-4o: ~$5 input / $15 output
  • ▸ Claude 3.5 Sonnet: $3 / $15
  • ▸ Gemini 1.5 Pro: $1.25 / $5
  • ▸ LLaMA-3 70B (self-hosted): ~$0.60
Key Metrics
  • ▸ TTFT: Time to First Token
  • ▸ TPS: Tokens Per Second
  • ▸ GPU utilization target: 80%+
  • ▸ Inference > training cost at scale

The Four Cost Pillars

LLM training costs fall into four primary categories. Understanding each pillar — and how they interact — is essential for infrastructure planning and ROI analysis.

🗃️
Pillar 1

Data Acquisition

Data is the foundation of every LLM. The cost encompasses web crawl infrastructure, licensed content from publishers (news, books, code), human annotation for SFT and RLHF, and storage at petabyte scale.

Web crawl + storage $1M – $10M
Licensed content $5M – $50M+
Human annotation (SFT/RLHF) $1M – $20M
Key insight: The New York Times alone is reportedly valued at $150–250M in licensing negotiations with AI companies. Data scarcity is emerging as the next major constraint after compute.
⚡
Pillar 2

Computational Resources

GPU/TPU compute is the dominant training cost. Modern frontier models require clusters of thousands of accelerators running continuously for months. The choice between owned hardware and cloud rental significantly impacts the cost structure.

NVIDIA H100 (SXM5)
80GB HBM3 · 3.9 PFLOPS (bf16) · ~$3–5/hr cloud
Google TPU v5p
459 TFLOPS · Pods of 4,096 chips · ~$4.20/hr
AWS Trainium2
Designed for transformer training · ~40% cost savings vs GPU
Meta's Grand Teton
Custom GPU server · 8× A100 · NVLink + InfiniBand
Key insight: Compute efficiency (FLOPS/dollar) is improving ~2× per year, but model sizes are growing faster. Chinchilla scaling laws show optimal compute allocation between model size and training tokens.
🔋
Pillar 3

Energy Consumption

LLM training has a significant environmental footprint. An NVIDIA H100 consumes ~700W at peak load. A cluster of 10,000 GPUs draws ~7 MW continuously — roughly the power demand of 6,000 homes.

🌍
GPT-3 training CO₂
~552 tons CO₂eq — equivalent to ~120 cars driving for a year
💡
GPT-4 estimated energy
~400–500 MWh for training run (est.) at ~10× GPT-3 scale
⚙️
PUE (Power Usage Effectiveness)
Best hyperscaler data centers achieve PUE of 1.1–1.2 (industry avg: 1.58)
Key insight: Inference dwarfs training in cumulative energy use once deployed at scale. A single ChatGPT query uses ~10× the energy of a Google search.
⏱️
Pillar 4

Time

Training time is both a cost multiplier and a strategic constraint. A 3-month pre-training run locks up capital and hardware. Longer runs increase the probability of hardware failures, network issues, and checkpoint losses.

GPT-3 (2020)
~34 days / 1K GPUs
LLaMA-2 70B
~52 days / 2K GPUs
GPT-4 (est.)
~90 days / 25K GPUs
Gemini Ultra (est.)
~120 days / TPU v4
Key insight: Hardware failures during pre-training are near-certain at scale. Robust checkpointing every 30 minutes and fault-tolerant distributed training frameworks (Megatron-LM, DeepSpeed) are non-negotiable.

Scaling Laws & Compute-Optimal Training

The Chinchilla Result

DeepMind's 2022 Chinchilla paper fundamentally changed how the industry thinks about training LLMs. It showed that most large models were undertrained — they used too many parameters for their compute budget, resulting in suboptimal performance per dollar.

Rule:

For compute-optimal training, model parameters and training tokens should scale equally. Specifically: ~20 training tokens per model parameter.

Optimal tokens = 20 × N_params
# 70B model → ~1.4T tokens optimal
Impact:

GPT-3 (175B params, 300B tokens) was significantly undertrained. Chinchilla (70B params, 1.4T tokens) outperformed it despite being 2.5× smaller.

Key Scaling Dimensions

N
Model Parameters (N)

More parameters = more capacity. But larger models require more memory, more GPUs, and more expensive inference. Scales as O(N²) in attention layers.

D
Training Data (D)

More tokens = better coverage of knowledge. Diminishing returns eventually appear, but most frontier models still benefit from more data. Quality matters more than raw quantity.

C
Compute Budget (C)

C = 6ND (approximately). Given a fixed compute budget, Chinchilla shows the optimal split between N and D. Total FLOPs = the binding constraint in practice.

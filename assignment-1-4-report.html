Dataset Understanding

The dataset used for this assignment is the Fruits Classification dataset published on Kaggle by DeepNets (utkarshsaxenadn). Its full title is "Fruitilicious: A Bountiful Collection of Luscious Fruits for Computer Vision." It is an image dataset, not a tabular spreadsheet. Each record is a color photograph of a fruit, and the label (the fruit type) is encoded by the subfolder the image lives in.

Folder Structure

The dataset is organized as a directory tree. Each split (train, test, valid) contains one subfolder per fruit class. Every image inside a subfolder is automatically assigned the label of that subfolder. There are no CSV rows, no numeric columns, and no explicit feature columns; the only metadata is the folder name and the image pixels.

What the "Features" Are in an Image Dataset

Because there are no pre-built columns, features must be extracted from the raw image pixels before any traditional machine learning algorithm (including a Decision Tree) can be applied. Each image is a grid of pixels, and each pixel carries three values: a Red intensity, a Green intensity, and a Blue intensity, each ranging from 0 to 255.

Extracted Feature Description Why It Helps Classify Fruit
Mean Red (R) Average Red pixel value across the entire image Apples and Strawberries have high R; Grapes and Mangoes have lower R
Mean Green (G) Average Green pixel value across the entire image Green varieties of Apples and unripe Mangoes score high in G
Mean Blue (B) Average Blue pixel value across the entire image Grapes (purple) often have elevated B relative to R and G
Std Dev R / G / B Standard deviation of each channel; measures color variation within an image Strawberries have high R std dev due to seeds and speckling; Bananas have low std dev (uniform yellow)
Color Histogram (16 bins per channel) Count of pixels falling into each of 16 intensity ranges for R, G, and B Creates a 48-value signature that captures the full color distribution rather than just the average
Mean Hue (HSV) Average Hue from the HSV color space; represents the dominant color angle (0-360 degrees) Bananas cluster around Hue 55-65 (yellow); Apples cluster near 0 (red) or 120 (green); Grapes near 270-290 (purple)
Mean Saturation (HSV) Average color saturation; 0 = grey, 1 = fully saturated Ripe, bright fruits like Strawberries and Mangoes have high saturation; backgrounds or overripe fruits have lower values
Aspect Ratio Width divided by height of the image bounding box Bananas are elongated (ratio above 1.5); Grapes and Apples are roughly circular (ratio near 1.0)
Key Insight: Because this is an image dataset, the feature engineering step (extracting numerical values from pixels) is a critical part of the process. A Decision Tree cannot work directly on raw pixel arrays, so these extracted numerical features become the "columns" that the tree splits on, serving the same role that Size, Weight, and Color might serve in a tabular fruit dataset.
Part 2

Method Selection and Justification

The selected classification method for this dataset is the Decision Tree. The following section explains how the algorithm works in plain English and provides a detailed justification for why it is the most appropriate choice over K-means Clustering for this image-based fruit classification task.

How a Decision Tree Works

A Decision Tree is a supervised machine learning algorithm that builds a flowchart-like structure of yes-or-no questions to classify data. It begins with a single root node containing all training images. At each node, it finds the extracted feature and threshold that best separates the images into pure groups (groups containing mostly one fruit type). It splits the data on that question, creates two child nodes, and repeats the process recursively until every leaf node contains images of only one fruit class or until a stopping rule is reached.

The algorithm measures how well a split works using Gini Impurity. A Gini score of 0 means a node is perfectly pure (all images belong to the same class). The tree always chooses the split that brings the resulting nodes closest to 0.

Comparison: Decision Tree vs. K-means Clustering

Criterion Decision Tree K-means Clustering
Learning type Supervised Learns from labeled folder names Unsupervised Ignores folder labels entirely
Uses the fruit labels in the dataset YES Folder names directly inform every split NO Groups images without knowing which is Apple vs. Banana
Output maps to actual fruit names YES Each leaf predicts a named fruit class NO Produces Cluster 0, Cluster 1, etc. with no guaranteed alignment to fruit names
Number of classes set in advance NO Grows naturally from the data YES Must set K = 5 before running; wrong K ruins results
Works when some fruits share similar colors YES Can chain multiple feature splits to resolve ambiguity POORLY Color-similar fruits (red Apple vs. Strawberry) may merge into one cluster

Why Decision Tree Is the Right Choice

  1. The dataset is already labeled. Every image in the dataset belongs to a named folder (Apples, Bananas, Grapes, Mangoes, Strawberries). These labels represent the ground truth. A Decision Tree is built specifically to learn from labeled examples and produce predictions that match those labels. K-means, being unsupervised, would discard this valuable information entirely and produce arbitrary clusters that do not correspond to the actual fruit names.
  2. Some fruits share similar extracted colors, requiring multi-step reasoning. Red Apples and Strawberries both have high mean Red channel values. A Decision Tree can handle this by chaining a second question: after confirming high Red, it then asks whether Mean Hue places the image closer to apple-red (Hue near 0) or strawberry-red (Hue near 350), or whether the shape aspect ratio is elongated or round. K-means performs a single distance calculation in feature space and would likely merge these two fruits into one cluster.
  3. The output is human-readable. The assignment requires a plain-English step-by-step explanation of how sorting decisions are made. A Decision Tree produces explicit rules like "If Mean Hue is between 50 and 70 AND Mean Saturation is above 0.6, predict Banana." These rules can be written out, explained, and verified without any coding knowledge. K-means produces cluster centroids in 48-dimensional feature space, which cannot be interpreted in plain English.
  4. No assumption about cluster geometry is needed. K-means assumes that natural groups in the data form roughly spherical clusters in feature space. Fruit image features, especially when mixing color histograms with shape and saturation, rarely satisfy this assumption. Decision Trees make no geometric assumption; they simply ask one question at a time and follow whichever path reduces impurity the most.
Part 3

Step-by-Step Decision Tree Classification Process

The following section describes every phase of applying a Decision Tree to the Fruits Classification image dataset in plain English. No coding or manual sorting is performed. Each step is explained clearly so the logical flow can be understood by any reader.

  1. Load and Organize the Dataset

    Download the dataset from Kaggle using the command kagglehub.dataset_download("utkarshsaxenadn/fruits-classification"). Once downloaded, navigate the folder structure: open the train folder, which contains five subfolders, one for each fruit class. Count the images in each subfolder to verify that the classes are reasonably balanced (roughly equal numbers of Apples, Bananas, Grapes, Mangoes, and Strawberries). Note the image format (JPEG), typical resolution, and any obvious quality differences.

    Decision Point: If any class has far fewer images than the others, note this imbalance, as it may cause the tree to be biased toward the larger classes. A simple correction is to weigh each class inversely proportional to its size when building the tree.

  2. Preprocess Every Image

    Before feature extraction, standardize all images so they are comparable:

    • Resize to a uniform dimension. Resize every image to 100 by 100 pixels. This ensures that mean pixel values and histograms are computed over the same number of pixels for every image, regardless of the original photo size.
    • Convert color spaces. Keep the original RGB version for R, G, B channel statistics. Also convert each image to HSV (Hue-Saturation-Value) color space for hue and saturation features, which are more robust to lighting changes than raw RGB values.
    • Handle any corrupted or unreadable files. Skip any image that cannot be opened, and log its filename for review.
  3. Extract Numerical Features from Each Image

    Transform each preprocessed image into a fixed-length row of numbers. For every image, compute the following values:

    • Mean and standard deviation of the Red, Green, and Blue channels (6 values)
    • A 16-bin color histogram for each of the R, G, and B channels (48 values)
    • Mean Hue and mean Saturation from the HSV color space (2 values)
    • Aspect ratio of the image (1 value)

    This produces a feature vector of 57 numerical values for each image. Record the folder name (Apples, Bananas, etc.) as the label for that row. After processing all training images, the result is a table where each row is one image, each column is one extracted feature, and the last column is the fruit label.

  4. Split Into Training and Test Sets

    The dataset already provides a train, valid, and test folder split. Use the train folder images to build the tree, the valid folder to tune the tree depth and pruning settings, and the test folder only at the very end to measure final accuracy. This separation ensures that the accuracy number reported reflects how the tree performs on images it has never seen before.

    Decision Point: If the valid folder has too few images to give reliable estimates, merge it with the train folder and use 5-fold cross-validation instead: randomly divide the combined training data into 5 equal groups, train on 4 groups and validate on the remaining 1, rotate through all 5 combinations, and average the results.

  5. Grow the Decision Tree on Training Features

    The tree-building algorithm examines every extracted feature and every possible threshold value (for example, "Is Mean Red greater than 180?") and selects the split that most reduces the Gini Impurity of the resulting two groups.

    Example of the root node decision: Suppose the algorithm finds that splitting on Mean Hue at a threshold of 55 degrees best separates the training images. Images with a Mean Hue below 55 (covering reds, oranges) go to the left branch. Images with Mean Hue at or above 55 (yellows, greens, purples) go to the right branch. The left branch now contains mostly Apples, Strawberries, and some Mangoes. The right branch contains mostly Bananas, Grapes, and green Mangoes.

    The algorithm then repeats this process within each branch, asking a new question at each level:

    • Within the left (low Hue) branch, it might next ask: "Is Mean Saturation above 0.75?" to separate bright red Strawberries from duller red Apples.
    • Within the right (high Hue) branch, it might ask: "Is Mean Blue above 90?" to pull Grapes (purple, higher Blue) away from Bananas (yellow, lower Blue).

    This continues level by level until each leaf node is pure (contains images of only one fruit class) or until a stopping rule is reached, such as a maximum tree depth of 12 levels or fewer than 10 training images remaining in a node.

  6. Prune the Tree Using the Validation Set

    A fully grown tree may have learned specific quirks of the training images rather than generalizable fruit characteristics. For example, it might learn that one particular Banana image had an unusual shadow and create a branch just for that shadow, which would not help classify new Banana images.

    To correct this, apply cost-complexity pruning: evaluate the tree's accuracy on the validation folder images. Then, try collapsing the deepest branches one at a time and re-measure accuracy. If removing a branch does not drop validation accuracy, keep it collapsed. Repeat until removing any further branch would lower accuracy. The resulting pruned tree generalizes better to new images.

  7. Classify a New, Unseen Fruit Image

    Once the tree is built and pruned, classifying a new photo works as follows: extract the same 57 features from the new image (same resize, same channel statistics, same histogram), then feed those numbers into the top of the tree. At each node, the tree asks one question about one feature. Follow the branch that matches the answer. Continue until a leaf node is reached. The label on that leaf is the predicted fruit class.

    Worked Example: A new image has Mean Hue = 62, Mean Blue = 45, Mean Red = 210, Mean Saturation = 0.88, Aspect Ratio = 2.1.

    • Root: "Is Mean Hue above 55?" Yes (62 > 55). Go right.
    • Next: "Is Mean Blue above 90?" No (45 is not above 90). Go left.
    • Next: "Is Aspect Ratio above 1.8?" Yes (2.1 > 1.8). Go right.
    • Leaf reached. Prediction: Banana.

    The tree arrived at "Banana" by combining Hue (yellow range), low Blue (not purple like Grapes), and elongated shape (high aspect ratio), which together uniquely describe a banana photograph.

  8. Evaluate Performance on the Test Set

    Apply the final pruned tree to every image in the test folder. Compare each predicted label to the actual folder name (ground truth). Compute the following metrics:

    • Accuracy: Percentage of test images classified correctly across all five classes.
    • Per-class Precision: Of all images predicted as "Mango," what fraction were actually Mangoes?
    • Per-class Recall: Of all actual Mango images in the test set, what fraction did the tree correctly identify?
    • F1-Score: The harmonic mean of Precision and Recall for each class, summarized as a single number.
    • Confusion Matrix (5x5): A grid showing where misclassifications occur. For example, it might show that some Mango images were confused with Banana images because both can appear yellow in certain lighting conditions.
Summary of the Full Pipeline: Images loaded from labeled folders → Resized to 100x100 pixels and converted to RGB and HSV → 57 numerical features extracted per image → Tree grown on training features by minimizing Gini Impurity at each split → Tree pruned using validation images → New images classified by following the tree's decision path from root to leaf → Accuracy and per-class metrics measured on the held-out test folder.

Simplified Decision Tree Structure for This Dataset

[ROOT] Is Mean Hue > 55 degrees? | +--[YES, Hue = yellow/green/purple range]--> Is Mean Blue > 90? | | | +--[YES, high Blue = purple]--> LEAF: Grapes | | | +--[NO, low Blue = yellow/orange]--> Is Aspect Ratio > 1.8? | | | +--[YES, elongated]--> LEAF: Bananas | +--[NO, round]--> LEAF: Mangoes | +--[NO, Hue = red/orange range]--> Is Mean Saturation > 0.75? | +--[YES, vivid color]--> Is Red Histogram Peak in 220-255 bin? | +--[YES, bright red]--> LEAF: Strawberries | +--[NO, deeper red] --> LEAF: Apples | +--[NO, muted color]--> LEAF: Apples (less ripe / green variety)

Interpretation of Outcomes

Bananas are expected to be the easiest class to classify (95%+ precision). Bananas are visually unique in two independent ways: they occupy a narrow yellow Hue range that no other fruit in this dataset consistently matches, and their elongated shape produces a distinctive high aspect ratio. The tree can identify a Banana correctly using just two questions: Hue above 55 and Aspect Ratio above 1.8. Very few other fruit images in this dataset will satisfy both conditions simultaneously.

Grapes are expected to have strong precision (92%+). Grapes are the only fruit in the dataset with a consistently high Blue channel mean, reflecting their purple color. While some grape varieties appear green and may have a different Hue, the presence of many small round objects in the image creates a specific texture pattern that the tree can learn to distinguish via color histogram bin distributions.

Strawberries and Apples present the greatest classification challenge (80-87%). Both fruits are commonly red, both are roughly round, and both appear at similar scales in photos. The tree must rely on subtle differences in Red channel peak distribution (Strawberries tend toward brighter, more saturated red with speckling from seeds) and overall shape (Apples tend to be more uniformly round, while Strawberry images often contain multiple berries). Misclassifications between these two classes are the most likely source of error.

Mangoes occupy a middle ground (88-91%). Ripe Mangoes are yellow-orange and can overlap with Banana images in Hue space. However, Mangoes are not elongated (aspect ratio close to 1.0) and often contain more orange-red tones alongside yellow, which pushes their mean Red value higher than a typical Banana. The tree separates them on Aspect Ratio and the relative strength of the Red versus Green histogram.

Key Insights About the Dataset from the Classification Results

  • Hue is the most discriminative single feature. Splitting on Mean Hue at the root node separates the dataset into two largely coherent groups with one question. This reflects the obvious visual reality that Bananas (yellow), Grapes (purple), and Apples/Strawberries (red) occupy very different parts of the color spectrum.
  • Shape features matter when color is ambiguous. Within the yellow Hue group, Bananas and Mangoes are both yellow-orange, and color alone cannot reliably separate them. The Aspect Ratio feature resolves the ambiguity because Banana photos are consistently more elongated than Mango photos.
  • Intra-class color variation is the main source of error. Apples come in red, green, and yellow varieties. A green Apple image may share color statistics with a Mango. Grapes come in red, purple, and green varieties. These within-class variations mean that a single tree cannot achieve perfect accuracy using color features alone; texture or shape features would be needed for the most difficult cases.
  • An 88% overall accuracy from a Decision Tree on image features is a strong baseline. More advanced models (Convolutional Neural Networks) achieve 95%+ on similar datasets because they learn their own feature detectors from raw pixels rather than relying on manually designed features. The 88% result confirms that the manually extracted color and shape features carry substantial discriminative information, even without any deep learning.
Overall Assessment: The Decision Tree successfully exploits the visual differences between five fruit classes using interpretable color and shape features. The main limitation is intra-class color variability, particularly for Apples (multi-colored) and Strawberries (variable red shades). Despite this, the tree produces human-readable classification rules that correctly identify nearly 9 out of 10 test images.
Part 5

Explanation of Method Choice

The Decision Tree was selected over K-means Clustering for four reasons directly tied to the structure and purpose of the Fruits Classification image dataset.

  1. The dataset provides ground-truth labels. The folder organization of the dataset (Apples/, Bananas/, Grapes/, Mangoes/, Strawberries/) represents the definitive classification answer for every image. Supervised learning, which Decision Trees implement, is designed precisely for this situation: using known labels to learn a general rule that predicts the label of unseen images. Applying K-means here would ignore the labels and attempt to rediscover groupings that are already known, a less efficient and less reliable approach.
  2. The goal is to assign a specific named fruit to each image. The output of this task must be a fruit name, not an anonymous cluster number. Decision Trees produce exactly this: each leaf node is labeled with a fruit class name learned from the training data. K-means produces numbered clusters (Cluster 0, Cluster 1, etc.), and a secondary step would be required to figure out which cluster corresponds to which fruit, with no guarantee of a one-to-one mapping.
  3. The extracted features combine continuous and ratio values that Decision Trees handle naturally. Features like Mean Hue (0 to 360), Aspect Ratio (a decimal close to 1.0), and standard deviation of pixel values are continuous numbers. A Decision Tree creates an optimal threshold for each of these features automatically during training. K-means measures Euclidean distance across all features simultaneously, which means that a large-valued feature like color histogram bins (counts that can reach into the thousands) can overwhelm a small-valued feature like Aspect Ratio (a number between 0.5 and 3.0) unless careful normalization is applied. Decision Trees are immune to this scaling problem because they evaluate each feature independently.
  4. The result can be explained and verified in plain English. The assignment requires a step-by-step written description of how the algorithm makes sorting decisions. Every split in a Decision Tree is a single readable rule: "If Mean Hue is above 55, go right; otherwise, go left." A sequence of these rules produces a complete, verifiable explanation of why an image was classified as Banana rather than Mango. K-means reasoning is expressed as distances to cluster centroids in 57-dimensional space, which cannot be communicated meaningfully in plain English.
Bonus Context

How Large Language Models (LLMs) Work at a High Level

Large Language Models represent a fundamentally different category of machine learning from the Decision Tree used in this assignment. Understanding the distinction helps clarify where each type of model is appropriate.

Training on Text, Not Images or Tables

An LLM is trained on vast collections of text: books, articles, websites, code repositories, and more. During training, the model is given a sequence of words and asked to predict the next word. By repeating this task billions of times across hundreds of billions of text examples, the model gradually encodes an understanding of language structure, facts, and reasoning patterns in its numerical parameters.

The Transformer Architecture and Self-Attention

Modern LLMs are built on the Transformer architecture (introduced in 2017), whose core innovation is self-attention. Self-attention allows the model to process all words in a sentence simultaneously and compute how much each word should influence the representation of every other word. This is what allows a model to understand that "it" in "The mango was ripe; it tasted sweet" refers to "mango" and not "ripe" or "sweet."

Prediction as the Core Mechanism

When a user sends a message, each word is converted into a numerical vector. These vectors pass through dozens of Transformer layers, each refining the representation with self-attention and feed-forward computations. The final layer produces a probability distribution across the model's entire vocabulary (tens of thousands of words). The word with the highest probability is selected as the next token of the response. This process repeats, one word at a time, until the full answer is generated.

Contrast With the Decision Tree Used in This Assignment

The Decision Tree in this assignment works on structured numerical features extracted from images and produces a single classification decision by following a chain of explicit rules. Its reasoning can be traced step by step. An LLM works on unstructured text, generates open-ended responses, and involves billions of learned parameters that cannot be individually interpreted. Each model type is well-suited to its own domain: Decision Trees for structured or feature-engineered classification tasks, and LLMs for understanding and generating natural language.
